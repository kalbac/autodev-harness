# M4 — Product Shell UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.
> This is a **review-only** UI build (presentation — NOT codex-gated per spec §8); each
> module is reviewed for behavior/visual fidelity, not adversarially gated. Steps use
> checkbox (`- [ ]`) syntax.

**Goal:** Replace the interim single-project shim with the real multi-project product
shell from the approved mockup (`docs/superpowers/specs/2026-07-03-s16-shell-mockup.html`):
a projects sidebar (last-5 runs + verdict seals + settings popover + theme), a
composer-first project Home, project-scoped `/p/:id/...` routes, a session-inspector
right rail, a New Project screen (folder browser + register form over the M3 endpoints),
and settings screens.

**Architecture:** `projectId` moves from the zustand store into the **router path**
(`/p/:id/...`); every project-scoped React-Query key and the `api` client gain a
`projectId` dimension so two projects never share cache; the WS invalidation filters by
the *active-route* projectId. The M3 endpoints (`GET /fs/dirs`, `POST /projects`,
`DELETE /projects/:id`) get typed `api` methods + queries. Visual modules follow the
mockup's tokens (already in `ui/src/styles.css` `@theme`).

**Tech Stack:** React 19 + Vite + TanStack Router/Query + zustand + Tailwind 4 (all
already in `ui/`), `lucide-react` icons, hand-rolled shadcn-idiom primitives in
`ui/src/components/ui/`. Build: `npm run build:ui` (root) → `dist/ui`; dev: `npm run dev:ui`.

**Branch:** `autodev/s17-m4-shell-ui` (already created; AGENTS.md tweak committed).

## Grounding facts (verified in-code 2026-07-04)

- **Current routing** (`ui/src/router.tsx`): flat routes `/`, `/runs/$runId`,
  `/tasks/$taskId`, `/board` under a `rootRoute` that renders `<AppShell><Outlet/></AppShell>`.
  `main.tsx` wraps the router in `ProjectGate` (auto-picks first project into the store).
- **`projectId` today**: lives in `useAppStore` (`ui/src/lib/store.ts`); `api.projectPath()`
  reads it via `useAppStore.getState().projectId`; `ws.ts` filters WS by the same store value.
- **api client** (`ui/src/lib/api.ts`): `getProjects` is daemon-global; every other method
  goes through `projectPath(subPath)`. M3 methods (`getFsDirs`, `postProject`, `deleteProject`)
  do NOT exist yet.
- **Query keys** (`ui/src/lib/queries.ts`): resource-first tuples, NO projectId dimension.
- **M3 endpoints live** and merged to main: `GET /fs/dirs?path=` → `{path,parent,entries:[{name,path,isGitRepo,isRegistered,isSymlink?}]}`;
  `POST /projects {path,name?,scaffold?,config?}` → 201 entry / 400|409 `{error,code}`;
  `DELETE /projects/:id` → 200 `{removed}` / 404.
- **Existing components to reuse/extend**: `ui/src/components/ui/VerdictSeal.tsx`,
  `Dot.tsx`, `StatusPill.tsx`, `Card.tsx`, `Button.tsx`, `Tabs.tsx`, `Feedback.tsx`;
  views `HomeView`, `RunView`, `TaskDetailView`, `BoardView`; `Sidebar`, `Inspector`,
  `AppShell`, `NewRunComposer`, `DigestStrip`, `EscalationCard`.
- **Design tokens** already in `ui/src/styles.css` (`--ink/panel/surface/line/text/muted/
  subtle/working/uncertain/broken/clean/accent`, fonts sans/mono/display). Mockup uses
  exactly these. **M5 adds the light set — not this module.**
- **Verdict → run mapping**: a run's verdict seal comes from its tasks' latest gate/critic
  outcome. A CLEAN-committed task has NO readable verdict artifact (gotcha
  `[ui/verdict-not-persisted]`) — the seal is best-effort from digest/escalation presence;
  do NOT invent a verdict endpoint. Runs with an escalated task → `uncertain`/`broken`
  seal via the existing `GET /escalations/:id`; an active run → `working`; otherwise
  `clean` is the optimistic default (documented in code).

- [ ] **Step 1: Router — add the `/p/:projectId` layout route.**

Rewrite `ui/src/router.tsx` so project-scoped views nest under a `projectRoute`
(`path: "/p/$projectId"`) whose component renders the shell + `<Outlet/>`, and add a
daemon-global index (`/`) that redirects to the first project (or the New Project screen
when none). Concretely:

```tsx
import {
  createRootRoute,
  createRoute,
  createRouter,
  Outlet,
  redirect,
} from "@tanstack/react-router";
import { AppShell } from "./components/AppShell";
import { HomeView } from "./views/HomeView";
import { RunView } from "./views/RunView";
import { TaskDetailView } from "./views/TaskDetailView";
import { BoardView } from "./views/BoardView";
import { NewProjectView } from "./views/NewProjectView";
import { api } from "./lib/api";

const rootRoute = createRootRoute({ component: () => <Outlet /> });

/** Daemon-global landing: pick the first project, else the New Project screen. */
const indexRoute = createRoute({
  getParentRoute: () => rootRoute,
  path: "/",
  beforeLoad: async () => {
    const { projects } = await api.getProjects();
    if (projects.length === 0) throw redirect({ to: "/new" });
    throw redirect({ to: "/p/$projectId", params: { projectId: projects[0]!.id } });
  },
});

/** Daemon-global New Project screen (no project selected). */
const newProjectRoute = createRoute({
  getParentRoute: () => rootRoute,
  path: "/new",
  component: () => (
    <AppShell>
      <NewProjectView />
    </AppShell>
  ),
});

/** Project-scoped layout: the shell + a nested outlet, all under /p/:projectId. */
const projectRoute = createRoute({
  getParentRoute: () => rootRoute,
  path: "/p/$projectId",
  component: () => (
    <AppShell>
      <Outlet />
    </AppShell>
  ),
});

const projectHomeRoute = createRoute({
  getParentRoute: () => projectRoute,
  path: "/",
  component: HomeView,
});
const runRoute = createRoute({
  getParentRoute: () => projectRoute,
  path: "/runs/$runId",
  component: RunView,
});
const taskRoute = createRoute({
  getParentRoute: () => projectRoute,
  path: "/tasks/$taskId",
  component: TaskDetailView,
});
const boardRoute = createRoute({
  getParentRoute: () => projectRoute,
  path: "/board",
  component: BoardView,
});

const routeTree = rootRoute.addChildren([
  indexRoute,
  newProjectRoute,
  projectRoute.addChildren([projectHomeRoute, runRoute, taskRoute, boardRoute]),
]);

export const router = createRouter({ routeTree, defaultPreload: "intent" });

declare module "@tanstack/react-router" {
  interface Register {
    router: typeof router;
  }
}
```

- [ ] **Step 2: A hook that reads the active-route projectId.**

Add `ui/src/lib/useProjectId.ts`:

```ts
import { useParams } from "@tanstack/react-router";

/**
 * The projectId from the active `/p/$projectId/...` route, or null on a
 * daemon-global route (`/`, `/new`). Replaces the old zustand `projectId`:
 * the URL is now the single source of truth for which project is selected.
 */
export function useProjectId(): string | null {
  const params = useParams({ strict: false }) as { projectId?: string };
  return params.projectId ?? null;
}
```

- [ ] **Step 4: Query keys + hooks gain the projectId dimension.**

In `ui/src/lib/queries.ts`, prefix every project-scoped key with the projectId and pass it
through the queryFn:

```ts
export const qk = {
  projects: ["projects"] as const,
  state: (p: string) => ["state", p] as const,
  runs: (p: string) => ["runs", p] as const,
  run: (p: string, id: string) => ["run", p, id] as const,
  runtimeFiles: (p: string, taskId: string) => ["runtime-files", p, taskId] as const,
  runtimeFile: (p: string, taskId: string, name: string) => ["runtime-file", p, taskId, name] as const,
  escalation: (p: string, id: string) => ["escalation", p, id] as const,
};

export const useProjects = () => useQuery({ queryKey: qk.projects, queryFn: api.getProjects });

export const useState = (p: string) => useQuery({ queryKey: qk.state(p), queryFn: () => api.getState(p) });
export const useRuns = (p: string) => useQuery({ queryKey: qk.runs(p), queryFn: () => api.getRuns(p) });
export const useRun = (p: string, id: string) =>
  useQuery({ queryKey: qk.run(p, id), queryFn: () => api.getRun(p, id) });
export const useRuntimeFiles = (p: string, taskId: string) =>
  useQuery({ queryKey: qk.runtimeFiles(p, taskId), queryFn: () => api.getRuntimeFiles(p, taskId) });
export const useRuntimeFile = (p: string, taskId: string, name: string | null) =>
  useQuery({
    queryKey: qk.runtimeFile(p, taskId, name ?? ""),
    queryFn: () => api.getRuntimeFile(p, taskId, name as string),
    enabled: name !== null,
  });
export const useEscalation = (p: string, id: string, enabled = true) =>
  useQuery({ queryKey: qk.escalation(p, id), queryFn: () => api.getEscalation(p, id), enabled });
```

- [ ] **Step 2: Query/mutation hooks in `queries.ts`:**

```ts
import { useMutation, useQueryClient } from "@tanstack/react-query";

export const useFsDirs = (path?: string) =>
  useQuery({ queryKey: ["fs-dirs", path ?? "__roots__"], queryFn: () => api.getFsDirs(path) });

export const useRegisterProject = () => {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (input: Parameters<typeof api.postProject>[0]) => api.postProject(input),
    onSuccess: () => void qc.invalidateQueries({ queryKey: qk.projects }),
  });
};

export const useDeleteProject = () => {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (id: string) => api.deleteProject(id),
    onSuccess: () => void qc.invalidateQueries({ queryKey: qk.projects }),
  });
};
```
